AMBench Leaderboard

The first standardized benchmark for AI agent memory systems

What is AMBench? A benchmark that tests the full memory lifecycle — encode, retrieve, consolidate, act — over 300 simulated incidents across 90 days. Unlike RAG benchmarks that only test retrieval, AMBench measures what agents actually need: temporal coherence, pattern recognition, and action relevance.

Metrics Explained

R@5Retrieval Recall@5 — Can the system find the right pattern type in top 5?
R@10Retrieval Recall@10 — Same but top 10 results
Cluster PurityAfter consolidation, how pure are the clusters vs ground truth patterns?
Temporal (avg days)Average age of top-5 results. Lower = more recent/relevant for agents.

Run AMBench

# Install
pip install imi-memory

# Run with defaults (300 incidents, 90 days)
python -m imi.benchmark

# Test your system with pure cosine (RAG baseline)
python -m imi.benchmark --rw 0.0 --name "my-system"

# JSON output for submission
python -m imi.benchmark --json > results.json

Submit Your System

To submit results to the leaderboard:

  1. Run python -m imi.benchmark --json > results.json
  2. Open an issue on GitHub with your results.json
  3. Include: system name, description, any modifications to the benchmark

We welcome submissions from any memory system: RAG variants, knowledge graphs, custom architectures. The goal is to establish a standard, not to win.

Submission Format

{
  "system": "your-system-name",
